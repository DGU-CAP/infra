GitHub에 새로운 이슈를 생성합니다.

## 진행 순서

1. 사용자에게 다음 정보를 확인하세요 (인자로 받은 경우 그대로 사용):
   - **이슈 제목** (간결하고 명확하게)
   - **이슈 유형**: `feat` (새 기능) / `fix` (버그) / `chore` (설정·유지보수) / `docs` (문서)
   - **작업 내용 설명**: 무엇을 왜 하는지

2. 아래 형식으로 이슈 본문을 작성하세요:

```
## 개요
(한 줄 요약)

## 작업 내용
- [ ] 세부 작업 1
- [ ] 세부 작업 2

## 관련 배경 / 이유
(이 작업이 필요한 이유)
```

3. `gh issue create` 명령어로 이슈를 생성하세요:
   - label은 이슈 유형과 동일하게 지정 (없으면 생략)
   - 생성 후 이슈 번호와 URL을 사용자에게 알려주세요

4. 이슈 생성 후 반드시 안내하세요:
   > 작업 시작 전에 `/new-branch` 로 이슈 번호 기반 브랜치를 만들고, 작업 완료 후 `/new-pr` 로 PR을 올려주세요.

## 브랜치 네이밍 규칙 (안내용)
`<type>/#<이슈번호>-<간단한-설명>` 형식 사용
예) `feat/#12-add-eks-cluster`, `fix/#7-subnet-cidr-overlap`
