현재 브랜치를 기준으로 GitHub Pull Request를 생성합니다.
PR은 반드시 연결된 이슈가 있어야 합니다.

## 진행 순서

1. **현재 상태 파악**
   - `git status`, `git log main..HEAD --oneline` 로 변경사항 확인
   - 브랜치명에서 이슈 번호 추출 (예: `feat/#12-add-eks` → `#12`)

2. **이슈 번호 확인**
   - 브랜치명에서 이슈 번호를 찾을 수 없으면 사용자에게 물어보세요
   - 이슈가 없다면 작업을 멈추고 안내하세요:
     > PR은 반드시 이슈와 연결되어야 합니다. 먼저 `/new-issue` 로 이슈를 생성해 주세요.

3. **PR 제목 작성**
   형식: `<type>: <작업 요약> (closes #<이슈번호>)`
   예) `feat: EKS 클러스터 Terraform 모듈 추가 (closes #12)`

4. **PR 본문 작성** (아래 템플릿 사용):

```
## 관련 이슈
closes #<이슈번호>

## 변경 사항
- 변경 내용 1
- 변경 내용 2

## 작업 내용 상세
(구현 방식, 주요 결정 사항 등)

## 체크리스트
- [ ] `terraform plan` 실행하여 의도한 변경만 포함되어 있음을 확인
- [ ] 리소스 태그 (Name, Environment) 누락 없음
- [ ] 민감 정보 (access key 등) 코드에 포함되지 않음
```

5. `gh pr create` 로 PR을 생성하세요:
   - base 브랜치: `main`
   - 생성 후 PR URL을 사용자에게 알려주세요

6. PR 생성 후 안내:
   > 팀원에게 리뷰를 요청하거나, `/review-pr <PR번호>` 로 AI 코드리뷰를 먼저 받아보세요.
